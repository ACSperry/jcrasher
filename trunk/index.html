Usage

The following describes two options for how to run JCrasher.

JCrasher automated with Ant

The jcrasher.xml Ant script compiles the testee sources, runs JCrasher on the testee, compiles the generated JUnit test files, runs JUnit on the generated test cases, and packages the results into a zip file, for example: examples-jcrasher-2.0.18-2006-10-08-1609.zip. The Ant script reads the textfile testees.txt and interprets every line as the location of a Java source file, relative to the src property in jcrasher.xml.
Example
cd examples
ant -f jcrasher.xml
ant -f jcrasher.xml clean

JCrasher directly from the command line

JCrasher expects fully qualified class names or package names as input parameters.
Example
cd examples
javac src\trivia\*.java -d .
java -cp ..\jcrasher-2.0.18.jar;. edu.gatech.cc.jcrasher.JCrasher trivia.DivByZero trivia.NullDeref
Now you need to compile and run the generated JUnit test cases.

Dependencies

JCrasher code links against (depends on) the following:

  • java-getopt-1.0.10.jar (GNU GetOpt) [LGPL].
    • Source release contains lib/java-getopt-1.0.10.jar (sources and binaries).
    • Binary release contains the binaries.
  • fb-crasher subset [Apache License 2.0].
    • Source release contains the sources.
    • Binary release contains the binaries.
  • JUnit 3.8.1 [Common Public License Version 1.0].
    • Not included.
    • Required to run the JCrasher test cases included in the source release.
    • Required to run any test cases generated by JCrasher.
